Module voice

Voice is a system to make the chat of Minetest more like a voice.

The only function that should be called from the client is activate.

Functions

abstruse (message, rate)
Abstruses the given message. That means that parts of it will be blanked, based on the rate.

Parameters:

  • message The message to abstruse.
  • rate The rate at which to abstruse the message, a value between 0 and 1, with 1 being everything abstrused.

Returns:

    The abstrused message.

Checks if the given distance is in the given range, considering the line of sight.

Parameters:

  • distance The distance.
  • range The range.
  • line_of_sight If there is line of sight.

Returns:

    true if it is in range.
invoke_message_callbacks (player, type, message)
Invokes all registered message callbacks.

Parameters:

  • player The player that is sending the message.
  • type The type of the message.
  • message The message to be send.

Returns:

    true if the message should be suppressed, and the second return value is the modified message.

Parameters:

    The muffled message.

Callback for if a chat message is send.

Parameters:

Returns:

    true if the message has been handled and should not be send.
random (rate)
Gets a random chance based on the given rate.

Parameters:

  • rate The rate. A number between 0 and 1, with 1 being always true.

Returns:

    true if there is a chance.

  • long The long command.
  • description The description of the command.
  • parameters The parameters to use.
register_global_chatcommand ()
Registers the chat command for global messaging.
register_on_message (callback)
Registers a callback that is invoked for every message that is send.

Parameters:

  • callback The callback that is to be registered, a function that accepts the player (Player object), the type of the message (TYPE_* constants), if the message is going to be send and the message itself. The callback can return two values, the first, a boolean that is true if the message should be suppressed, and the second being a message to be sent instead. The last callback can determine if the message should actually be send or not.
speak (speaking_player, message, parameters)
Speaks the given message.

Parameters:

  • message The message that is spoken.
  • parameters The speak parameters, like talk_parameters.
